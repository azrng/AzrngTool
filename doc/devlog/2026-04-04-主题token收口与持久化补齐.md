# 2026-04-04 主题 token 收口与持久化补齐

## 本次目标
- 统一主题 token 入口，避免应用层继续同时合并新旧两套 design tokens
- 清理数据库工作台里残留的硬编码颜色与阴影
- 补全状态色在亮色与暗色主题下的语义资源
- 为主窗口主题切换增加本地持久化，并让开关状态可回显

## 核心改动
- 将 `AzrngTools/Styles/DesignTokens.axaml` 作为应用唯一 token 入口，并在其中合并旧字体与间距兼容资源
- 在统一 token 中补齐状态色、叠层、危险边框、Hero 半透明层与阴影资源，覆盖亮色与暗色主题
- 将 `ToastNotification` 与 `LoadingOverlay` 切回主题资源，去掉硬编码背景、边框、文字色和阴影
- 将 `Misc.axaml` 中残留的 Excel 徽标、概览 Hero 玻璃层与图块颜色收口到主题资源
- 新增 `IThemePreferenceService` / `ThemePreferenceService`，在应用初始化时恢复主题偏好，并在主窗口切换时保存
- 给主窗口主题切换开关补齐 `IsChecked` 绑定，保证恢复主题后界面状态和实际主题一致
