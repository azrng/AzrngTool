# 2026-04-03 数据库模块文本编码统一

## 本次目标
- 统一数据库模块及相关宿主文件的文本编码格式，解决 Windows 默认 PowerShell 下中文源码显示为乱码的问题。
- 保持业务逻辑与可见文案不变，只做编码层面的稳定化处理。

## 核心改动
- 核对数据库模块相关 `.cs` 与 `.axaml` 文件的编码状态，确认数据库模块主体文件均为 `UTF-8 无 BOM`，而仓库内部分中文文档已使用 `UTF-8 with BOM`。
- 将以下范围内的数据库模块源码统一转为 `UTF-8 with BOM`：
  - `AzrngTools/ViewModels/Database`
  - `AzrngTools/Views/Database`
  - `AzrngTools/Controls/Database`
  - `AzrngTools/Services/Database`
  - `AzrngTools/Models/Database`
  - `AzrngTools/Converters/Database`
  - `AzrngTools/Utils/Database`
- 同步将与数据库模块直接相关、且含中文注释/文案的宿主文件转为 `UTF-8 with BOM`：
  - `AzrngTools/App.axaml.cs`
  - `AzrngTools/ViewModels/MainWindowViewModel.cs`
- 转换后再次验证数据库工作台核心 XAML 文本可在当前终端环境下正常显示。

## 风险或遗留项
- 本轮只统一了数据库模块及直接相关宿主文件的编码，仓库其余历史文件仍可能存在 `UTF-8 无 BOM` 与 `UTF-8 with BOM` 混用情况。
- 本轮没有改动业务逻辑与可见交互，只做编码层稳定化；数据库连接、浏览、导出等功能仍未做逐页手工回归。
