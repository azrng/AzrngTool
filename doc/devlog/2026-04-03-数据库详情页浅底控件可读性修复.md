# 2026-04-03 数据库详情页浅底控件可读性修复

## 本次目标
- 修复数据库工作台详情页在暗色主题下若干浅底控件可读性差的问题
- 消除表详情头部按钮、状态标签和行内操作按钮的突兀白底

## 核心改动
- 将 `Button.toolbarAction` 从旧的 `Brush.Background.Subtle` / `Brush.Info.*` 体系切换到 `Brush.Panel.*` 与 `Brush.Accent.*`
- 将 `Button.inlineAction` 的底色、边框和前景色统一到当前 Accent token，避免表格行内“编辑”按钮出现白底浅字
- 将 `Border.statusPill` 与 `TextBlock.statusPillText` 改为 `Brush.Accent.Soft` / `Brush.Accent.Border` / `Brush.Accent.Strong`

## 修改文件
- `TASK.md`
- `AzrngTools/Assets/Styles/Controls/Buttons.axaml`
- `AzrngTools/Assets/Styles/Controls/Misc.axaml`

## 校验情况
- 已执行：`dotnet build AzrngTools.sln`
- 结果：通过，0 warning，0 error

## 风险或遗留项
- 本轮修的是数据库详情页直接依赖的共享样式，没有继续全面清理所有旧 `Brush.Neutral.*` / `Brush.Info.*` 按钮样式
- `ConnectionDialog.axaml` 等数据库模块其他弹窗仍残留旧浅色硬编码，后续可继续按同样方式分批收口
