# 数据库工作台遗留性能专项

## 本次目标
- 继续处理数据库工作台性能遗留项：对象树懒加载、搜索防抖与日志队列。

## 核心改动
- 数据库对象树初次加载只构建 Schema 骨架，表、视图、存储过程与 MySQL 函数改为选中分组时懒加载。
- 对象浏览器搜索输入增加 250ms 防抖，减少连续输入时的递归过滤与节点克隆开销。
- 日志服务从每条日志 `Task.Run` 改为单消费者队列串行写入，降低高频日志下的线程池压力。
- 补充对象树骨架、防抖器和日志队列的单元测试。
