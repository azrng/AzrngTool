# 2026-04-08 稳定发布配置切回非 AOT

## 本次目标
- 将项目默认 Release 发布策略从 `AOT + Trim` 切回更稳妥的桌面发布方案
- 保留关于页更新中心能力，避免继续把正式发版稳定性绑定在 Native AOT 兼容性上

## 核心改动
- 调整 `AzrngTools.csproj` 的 Release 发布配置，改为 `SelfContained + PublishSingleFile + PublishReadyToRun`，并显式关闭 `PublishAot` 与 `PublishTrimmed`
- 移除仅服务于 AOT/Trim 发布链路的 `TrimmerRootDescriptor`、`Costura.Fody` 与 `PublishAotCompressed` 配置
- 统一本地发布入口，`Publish.bat`、`Publish - Win.bat` 改为调用根目录 `publish.ps1`
- 更新 `publish.ps1` 与 GitHub Actions `release.yml`，发布前清理输出目录，并显式使用非 AOT 的稳定发布参数
- 更新 `README.md` 中的本地发布命令说明，补充当前默认发布策略说明
- 保留 `AppUpdateService` 中针对 GitHub Release 解析的 source-generated JSON 元数据实现，避免未来如再次尝试 AOT 时立刻踩回更新中心序列化问题

## 校验情况
- 已执行：`dotnet build AzrngTools.sln`
- 已执行：`dotnet publish AzrngTools\\AzrngTools.csproj -c Release -r win-x64 --self-contained true -p:PublishAot=false -p:PublishTrimmed=false -p:DebugType=None -p:DebugSymbols=false`
- 结果：构建与发布均通过，发布输出位于 `AzrngTools/bin/Release/net10.0-windows/win-x64/publish/`
