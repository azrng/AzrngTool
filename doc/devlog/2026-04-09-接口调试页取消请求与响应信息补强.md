# 2026-04-09 接口调试页取消请求与响应信息补强

## 本次目标
- 移除接口调试页底部单独的工作状态区。
- 去掉 Path 参数配置区。
- 在响应区显示最终发送的完整请求 URL。
- 为请求发送过程补上可见加载态，并支持用户主动取消。

## 核心改动
- 精简请求配置区，只保留 Query、Headers 和 Body 三类输入，不再提供 Path 参数编辑入口。
- 调整请求执行链路，响应快照中补充 `FinalUrl`，响应区新增完整请求 URL 展示卡片。
- 为请求发送增加进行中状态和取消命令，请求过程中显示进度条与取消按钮，并在取消后返回可见反馈。
- 移除页面底部工作状态卡片，把过程反馈收口到响应区和必要的消息提示中。

## 风险或遗留项
- 当前“取消请求”依赖 `CancellationToken` 中断 `HttpClient.SendAsync`，实际取消反馈仍建议对慢接口手工走一遍。
- 若应用运行中长期占用输出目录，常规 `dotnet build` 仍可能出现复制警告；关闭正在运行的桌面程序后可恢复到无警告构建。
