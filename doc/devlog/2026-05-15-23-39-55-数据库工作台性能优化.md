# 数据库工作台性能优化

## 本次目标
- 降低数据库工作台在大结果集查询和表详情加载时的性能风险。

## 核心改动
- 为 SQL 查询页增加预览查询限流，默认将可限制的 `SELECT` 查询收口到最多 1000 行。
- 表详情加载时不再默认执行精确 `COUNT(1)`，避免打开大表详情时触发高成本全表统计。
- 行数未加载时显示“未加载”，避免把未知行数误展示为 0。
- 补充查询限流和行数显示的单元测试。

## 风险或遗留项
- SQL Server 当前仅对普通 `SELECT` 注入 `TOP`，复杂 `WITH` 或 `SELECT DISTINCT` 暂不自动改写，避免误改用户 SQL。
- 本次未改造数据库对象树懒加载、搜索防抖和日志队列，可作为后续性能专项继续处理。
