# 版本号同步为 UTC 四段规则

## 本次目标
- 参考 `ApixPress` 将 `AzrngTools` 的自动版本号生成规则同步为 UTC 年月日小时四段格式。

## 核心改动
- `Directory.Build.props` 默认版本从本地日期 `yyyy.M.d` 改为 UTC 时间 `yyyy.M.d.HH`。
- `.github/workflows/release.yml` 在 `VERSION=auto`、为空或不存在时使用 UTC 四段版本生成发布元数据。
- 更新 `README.md` 的自动发布与版本号规则说明。

## 修改文件
- `Directory.Build.props`
- `.github/workflows/release.yml`
- `README.md`
- `TASK.md`
- `doc/devlog/2026-04-28-22-00-00-版本号同步为UTC四段规则.md`

## 校验情况
- 已执行：`dotnet msbuild AzrngTools\AzrngTools.csproj -getProperty:Version -getProperty:InformationalVersion -getProperty:AssemblyVersion -getProperty:FileVersion -getProperty:IncludeSourceRevisionInInformationalVersion`
- 校验结果：默认自动解析为 UTC 四段版本，示例为 `2026.4.28.13`，且 `IncludeSourceRevisionInInformationalVersion=false`。
- 已执行：`dotnet msbuild AzrngTools\AzrngTools.csproj -p:AzrngToolsLegacyVersion=2026.4.9 -getProperty:Version -getProperty:InformationalVersion -getProperty:AssemblyVersion -getProperty:FileVersion`
- 校验结果：显式版本覆盖后输出为 `2026.4.9 / 2026.4.9.0`。
- 已执行：`dotnet build AzrngTools.sln -c Release -v minimal`
- 已执行：`dotnet test AzrngTools.sln -c Release -v minimal --no-build`
- 已执行：`dotnet publish AzrngTools\AzrngTools.csproj -c Release -r win-x64 --self-contained true -o publish\win-x64-version-rule-test -p:AzrngToolsVersionDate=2026.4.28.13 -p:PublishAot=false -p:PublishTrimmed=false -p:DebugType=None -p:DebugSymbols=false -v minimal`
- 校验结果：发布出的 `AzrngTools.exe` 的 `ProductVersion` 与 `FileVersion` 均为 `2026.4.28.13`，启动 5 秒后进程仍存活，随后已关闭。

## 风险或遗留项
- 自动版本号中的小时为 UTC 小时，不再等同本地时间小时；这是为保证本地与 CI 生成规则一致的约定。
